중국 그린댐

**개요**

2009년 5월, 중국 공업정보화부(MIIT)는 중국 내에서 판매되는 모든 컴퓨터 시스템에 유해 차단 소프트웨어인 '그린 댐 유스 에스코트'를 의무적으로 설치하도록 하는 조치를 발표했습니다. 원래 목적은 청소년을 음란물·폭력물 등 유해 콘텐츠로부터 보호한다는 것이었습니다.

**논란이 된 이유**

- **검열 우려**: 유해물 차단이라는 명목 외에도, 중국 정부가 반체제적 사상으로 금지한 파룬궁 관련 단어 등을 차단어 목록에 포함시켰고, 사용자 감시 목적으로 악용될 수 있다는 비판이 제기되었습니다.
- **보안 결함**: 보안 연구자들이 프로그램 자체에 심각한 취약점이 있다고 지적했습니다.
- **지적재산권 분쟁**: 미국의 한 소프트웨어 업체가 그린 댐이 자사에서 훔친 프로그래밍 코드를 사용했다며 소송을 준비한다고 밝히기도 했습니다.

**시행 연기 및 철회**

- 중국 정부는 2009년 7월 1일 시행 예정이었던 의무 탑재 조치를 시행 1시간 전에 전격 연기했습니다.
- 이후 중국 정부는 일반 사용자용 PC에는 그린 댐 설치를 강제하지 않는다고 발표하며 사실상 전면 철회했고, 공립학교와 인터넷 카페 등 공공장소 컴퓨터에만 설치할 계획이라고 입장을 축소했습니다.
- 하지만 학교 현장에서도 기존 수업용 소프트웨어 및 학교 행정 시스템과의 호환성 문제로 그린 댐을 제거하는 사례가 이어졌다고 알려졌습니다.

**의의**

이 사건은 중국 정부의 인터넷 통제 시도가 국내외적으로 얼마나 강한 반발에 부딪힐 수 있는지를 보여준 대표적 사례로 꼽히며, 이후 중국의 인터넷 검열은 '만리방화벽(Great Firewall)'을 통한 네트워크 단의 통제로 방향을 틀게 됩니다.

쿠키와 로컬스토리지

웹 브라우저에 데이터를 저장하는 두 가지 대표적인 방식입니다. 비교해서 정리해 드릴게요.

## 핵심 비교

| 구분 | 쿠키 (Cookie) | 로컬스토리지 (LocalStorage) |
|---|---|---|
| **용량** | 약 4KB | 약 5~10MB |
| **만료** | 설정한 만료일까지 (또는 세션 종료 시) | 명시적으로 삭제하지 않는 한 영구 |
| **서버 전송** | 모든 HTTP 요청에 자동으로 포함되어 전송됨 | 전송되지 않음 (JS로만 접근) |
| **접근 위치** | 클라이언트 + 서버 둘 다 | 클라이언트(JS)만 |
| **API** | `document.cookie` (문자열 파싱 번거로움) | `localStorage.setItem/getItem` (간편) |

## 사용 예시

**쿠키**
```javascript
// 설정 document.cookie = "userId=hye3tree; expires=Fri, 31 Dec 2026 23:59:59 GMT; path=/";

// 읽기 (직접 파싱 필요) document.cookie; // "userId=hye3tree; theme=dark"
``` **로컬스토리지**
```javascript
// 설정
localStorage.setItem("theme", "dark");
// 읽기
localStorage.getItem("theme"); // "dark"
// 삭제
localStorage.removeItem("theme");
localStorage.clear(); // 전체 삭제
``` ## 언제 무엇을 쓰나

- **쿠키**: 서버가 요청마다 알아야 하는 정보 → 예) 로그인 세션 토큰, 인증 정보
- **로컬스토리지**: 서버가 몰라도 되는, 브라우저에서만 쓰는 정보 → 예) UI 테마 설정, 임시 폼 데이터, 캐시된 데이터
## 관련 개념 (같이 알아두면 좋은 것)
- **세션스토리지(SessionStorage)**: 로컬스토리지와 API는 동일하지만, **탭을 닫으면 사라짐** (같은 탭 내에서만 유지)
- **보안 이슈**: 로컬스토리지는 JS로 자유롭게 접근 가능해서 **XSS 공격**에 취약함 → 민감한 인증 토큰(JWT 등)을 로컬스토리지에 저장하는 건 보안상 권장되지 않고, `HttpOnly` 쿠키를 쓰는 게 더 안전하다는 논의가 많음 (전에 작업하신 예약 앱 XSS 대응과도 연결되는 부분입니다)

웹콘솔로그 남기기

브라우저에서 개발자 도구(DevTools)의 **콘솔(Console) 탭**을 말씀하시는 것 같습니다.
**여는 방법**
| 브라우저 | 단축키 |
|---|---|
| Chrome / Edge | `F12` 또는 `Ctrl+Shift+J` (Mac: `Cmd+Option+J`) |
| Firefox | `F12` 또는 `Ctrl+Shift+K` (Mac: `Cmd+Option+K`) |
| Safari | 개발자 메뉴 활성화 후 `Cmd+Option+C` (환경설정 → 고급 → "메뉴 막대에서 개발자용 메뉴 보기" 체크 필요) |
열리면 상단 탭 중 **"Console"**을 클릭하면 됩니다.
**로그를 남기는 방법 (JavaScript 코드에서)**
```javascript
console.log("일반 로그");
console.warn("경고 로그");
console.error("에러 로그");
console.info("정보성 로그");
console.table(배열또는객체); // 표 형태로 출력
console.group("그룹명"); // 로그 묶기
console.log("하위 로그");
console.groupEnd();
``` **Right-click 메뉴로도 접근 가능**: 페이지에서 우클릭 → "검사(Inspect)" → Console 탭
